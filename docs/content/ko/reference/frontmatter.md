---
title: Frontmatter
description: lily-pad가 마크다운 파일 상단에서 읽는 모든 필드.
section: 글쓰기
order: 2
---

모든 페이지는 YAML frontmatter 블록으로 시작합니다. `title`과 `description`은 필수이고 나머지는 선택입니다.

```md
---
title: 내 페이지
description: 이 페이지가 다루는 내용.
navLabel: 짧은 이름
section: 가이드
order: 3
---
```

## 필드

| 필드          | 필수 | 역할                                                       |
| ------------- | ---- | ---------------------------------------------------------- |
| `title`       | ✅   | 페이지 제목이자 브라우저 탭 제목                           |
| `description` | ✅   | 제목 아래 부제이자 메타 설명                               |
| `navLabel`    | —    | 사이드바용 짧은 라벨 (기본값은 `title`)                    |
| `section`     | —    | 사이드바 그룹 제목 — 로케일마다 다르게 지을 수 있음        |
| `order`       | —    | 사이드바 정렬 순서 (작을수록 먼저)                         |
| `links.source`| —    | 제목 아래 "Source ↗" 배지 추가                            |
| `links.api`   | —    | 제목 아래 "API ↗" 배지 추가                               |

## 사이드바가 만들어지는 방식

사이드바는 콘텐츠 폴더에서 생성됩니다 — 관리할 설정 파일이 없습니다:

1. 로케일 루트의 파일들이 첫 그룹이 됩니다.
2. 각 하위 폴더가 그룹이 되고, `section`(없으면 폴더명)이 제목이 됩니다.
3. 그룹 안에서는 `order`로 정렬됩니다.
